# Security Policy

## Supported Versions

At minimum, the latest `main` branch is supported for security fixes.

## Reporting a Vulnerability

Please report vulnerabilities privately.

- Preferred: open a private security advisory on GitHub (if enabled).
- Alternative: email the maintainer directly at `arshadali8376@gmail.com`.

Do not open a public issue for security vulnerabilities.

For latest contact details, see `SUPPORT.md`.

## What to Include

- Type of vulnerability
- Affected component/file
- Reproduction steps or proof of concept
- Potential impact
- Suggested remediation (optional)

## Response Timeline

- Initial acknowledgment: within 72 hours
- Triage decision: within 7 days
- Fix plan/mitigation update: as soon as validated

## Disclosure

Please allow time for investigation and patching before public disclosure.
Once fixed, maintainers may publish a security note with impact and remediation details.
